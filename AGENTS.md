# AGENTS.md

## Project

Max & Victoria wedding website. Single-page scroll, bilingual (English + Ukrainian), deployed on Vercel. Under 50 guests, destination wedding in Portugal. Physical and digital invitations go out simultaneously.

**Wedding**: September 13, 2026 at 1:00 PM
**Venue**: Vinha Boutique Hotel, R. Fonte da Vinha 383, 4430-487 Vila Nova de Gaia, Portugal
**Couple**: Max (Maksym) & Victoria

## Tech Stack

- Next.js 16.2.6 (App Router, React 19)
- TypeScript (strict)
- Tailwind CSS v4 (inline theme via `@theme inline`)
- shadcn/ui (8 components: Accordion, Button, Input, Textarea, Label, RadioGroup, Select, Sheet)
- next-intl 4.12.0 (English + Ukrainian)
- Framer Motion (scroll-reveal animations)
- Resend (RSVP email delivery)
- pnpm package manager

## Architecture

Single route at `/[locale]/page.tsx` composes 7 scroll sections. Navigation uses anchor links (`#hero`, `#our-story`, etc.) with smooth scrolling.

### Section Order

1. **Hero** (`#hero`): couple names, date, countdown, location, background photo
2. **Our Story** (`#our-story`): couple narrative (content TBD)
3. **Schedule** (`#schedule`): 7-block wedding day timeline
4. **Travel & Porto** (`#travel`): airport, accommodation, Porto guide + YouTube embeds
5. **Gallery** (`#gallery`): couple photos grid + guest photo album link
6. **FAQ** (`#faq`): accordion Q&A (dress code, kids, ceremony language, weather)
7. **RSVP** (`#rsvp`): form with dietary restrictions, email-only via Resend

## File Organization

```
src/
  components/
    ui/          shadcn primitives (generated by CLI, don't hand-edit)
    layout/      page chrome: navbar, footer, language toggle, sticky CTA, theme configurator
    sections/    the 7 scroll sections (hero, story, schedule, travel, gallery, faq, rsvp)
    shared/      reusable composites: countdown, map-embed, rsvp-form, section-wrapper, youtube-embed
  config/        site constants (wedding date, venue), theme definitions, navigation section IDs
  hooks/         custom React hooks (use-active-section, use-countdown, use-theme)
  lib/           utilities: cn() helper, email client, font config
  types/         shared TypeScript types (rsvp, theme, i18n)
  styles/        design tokens CSS (canonical source, imported by globals.css)
  actions/       server actions (RSVP submission)
  i18n/          next-intl routing, navigation, request config
  app/           Next.js App Router (layouts, pages, error boundaries)
```

## Design System

Full spec in `docs/design-system.md`. Key rules:

### Typography

- **Display**: Cormorant Garamond, weights 300 and 400 only. For headings 22px+ only.
- **Body**: Montserrat, weights 400 and 500 only. 16px body text, never smaller on mobile.
- **Mono**: system monospace for temporal data (countdown, dates, times).
- Never use weight 600+. Never use bold. The lightness is the elegance.

### Visual

- Zero shadows on everything. Depth comes from color contrast only.
- Zero gradients (except photo overlay scrims for text legibility).
- Photos never have border-radius. Aspect ratios: 4:5 portrait, 16:9 landscape, 1:1 grid.
- No emoji as decorative elements.
- No center-aligned body text. Only hero headlines, date, and countdown are centered.
- No dark mode. A wedding invitation is cream paper with dark ink.

### Animations

- Entrance: opacity + translateY, 200-500ms, ease-out. One animation per section.
- No bounce, no spinning, no parallax on mobile.
- Hero stagger sequence: 200ms, 500ms, 1000ms, 1300ms, 1800ms delays.

### Spacing

- 8pt grid base unit.
- Section vertical padding: `clamp(64px, 10vw, 120px)`.
- Container max-width: 1120px.
- Touch targets: 44px minimum on mobile.

## Theme System

Two data attributes on `<html>` control the visual system:

### `data-preset="natural|editorial"`

Controls component shapes (input borders, button fills, radii).

- **Natural**: full-border inputs, sage-filled CTAs, 4-6px radius, surface bg cards
- **Editorial**: bottom-border inputs, outlined CTAs, near-zero radius, alternating section bgs

### `data-palette="sage|rose|champagne|terra"`

Controls accent colors, borders, muted text.

- **Sage Green**: oklch(58% 0.09 155) / botanical, muted
- **Rose Gold**: #d4a88c / romantic, soft metallics
- **Champagne**: #c9a96e / sunlit, Mediterranean warmth
- **Terracotta**: #c08b6e / earthy, editorial

Both attributes are set by the ThemeConfigurator component (dev-only, bottom-left floating panel) and persisted to localStorage.

To add a new palette: add CSS variable overrides in `styles/tokens.css` under `[data-palette="new-name"]`, add the palette config to `config/theme.ts`, and add a swatch to the ThemeConfigurator.

## i18n

**Locales**: `en` (English, default), `uk` (Ukrainian)
**Messages**: `messages/en.json`, `messages/uk.json`

Section-based namespaces:

```
meta, nav, hero, ourStory, schedule, travel, gallery, faq, rsvp, footer
```

To add a translation: add keys to both en.json and uk.json with identical structure. Use `useTranslations("namespace")` in components.

All user-facing text must be in message files. Never hardcode strings in components.

## Conventions

- **Server Components by default.** Only add `"use client"` for interactive elements (forms, timers, menus, theme switcher).
- **Constants in config/.** Wedding date, venue, emails live in `config/site.ts`. Never hardcode in components.
- **shadcn components in ui/.** Generated by CLI. Customize via CSS custom properties in `styles/tokens.css`, not by editing the component files directly.
- **All text in i18n messages.** No hardcoded strings. Both languages must stay in sync.
- **Framer Motion for scroll animations** via the `section-wrapper.tsx` component.
- **No unnecessary abstractions.** This is a small site for under 50 guests. Keep it simple.

## Key Decisions

- **Email-only RSVP**: no database. Under 50 guests, email via Resend is sufficient.
- **External photo album**: guest photos via Google Photos or iCloud shared album link, not custom upload.
- **No gift registry**: no mention of gifts on the site.
- **No dark mode**: design system explicitly prohibits it.
- **Vinha Boutique Hotel is the venue, not an accommodation recommendation.** It is expensive. Do not recommend it for guest stays.
- **Kids and plus-ones welcome.**
- **Ceremony is bilingual** (English + Ukrainian).
